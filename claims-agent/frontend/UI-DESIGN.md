# 案件详情页 UI 设计稿 — Agent + 人工协作模式

## 设计原则

1. **一屏纵览** — 案件全貌、进度、问题、决策在一个页面内可见
2. **Agent 透明** — 每一步推理过程可视化，规则矩阵可钻取
3. **人机协作** — 对话式介入面板，随时纠偏/补充/暂停
4. **折叠优先** — 参考传统系统 accordion 模式，信息密度高但不拥挤
5. **红绿灯语义** — 全局统一 🟢通过/🔴拒赔/🟡补材/🔵转人工/⚪待执行

## 逐区域详细规格

### 1. TOP BAR（顶部状态栏）

```
高度: 56px | 背景: #fff | 底部边框: 1px solid #e8e8e8

┌──────────────────────────────────────────────────────────────┐
│ OLD-PROJ001-SP-20240320-000001  张三  诊断:多发性骨髓瘤       │
│ [🔄 审核中] [🔥紧急] [⏱ SLA: 剩余 4.2h]    [执行审核] [撤销] │
└──────────────────────────────────────────────────────────────┘
```

- 案件号 (14px bold)
- 被保险人 (13px)
- 诊断 (13px, 灰色)
- 状态标签 (11px, 彩色圆角)
- 优先级 (🔥 紧急/普通)
- SLA 倒计时 (正常=绿色 / 预警=橙色 / 超时=红色)
- 操作按钮 (右对齐)

### 2. 保单标签栏

```
高度: 40px | 背景: #fafafa | 底部边框: 1px solid #e8e8e8

保单: [P001 好医保·癌症特药险 ✅] [P002 悠臻保（特药责任） 🔄] [+ 查看全部]
```

- 每个保单一个 pill 标签
- ✅ = 审核通过 | 🔄 = 审核中 | ⚪ = 待执行

## 交互规范

### 折叠/展开
- 所有卡片默认展开, 可点击标题折叠
- 折叠状态显示摘要 (如: "L2 内容审核 8/10 通过")
- localState 管理, 不需要后端

### 保单切换
- 点击保单标签 → 左右栏内容联动切换
- 左栏: 该保单对应的诊断/用药/附件
- 右栏: 该保单的执行进度/规则矩阵/理算

### 附件查看器
- 点击缩略图 → 全屏模态框
- 左侧缩略图列表, 右侧大图预览
- 工具栏: 上一个/下一个/旋转/下载/OCR文字切换
- OCR 文字: 从 `if_attachment_content.response_content` 获取

### Agent 协作
- 输入框支持多行文本
- "发送并继续" → Agent 收到指令, 继续执行
- "暂停" → Agent 暂停当前 Phase, 等待进一步指令
- "转人工" → 标记案件为 transferToManual, 分配审核组

### 响应式
- ≥1024px: 左右双栏 (40% + 60%)
- 768-1023px: 单栏, 左栏在上, 右栏在下
- <768px: 移动端, 全部单栏, 折叠默认收起
